# 0.1 支持范围

本文档用于固化 `0.1` 版本的发布边界。仓库当前已经具备多数据库、多缓存和多存储能力，但 `0.1` 不会把所有组合都当成正式承诺。

从这个阶段开始：

- 这份文档是 `0.1` 支持级别的单一真源
- README、数据库运维文档、页面文案、CI 覆盖范围都应与这里保持一致
- 新增功能默认不再进入 `0.1` 范围，除非它是明确的 release blocker

## 支持级别定义

### GA

适用于 `0.1` 正式版默认推荐路径。

要求：

- README、运维文档、CI 和页面行为口径一致
- 有稳定的主链路测试和回归覆盖
- 升级、备份、恢复语义已经收敛
- 允许作为默认生产建议写进首页文档

### Beta

功能可用，但不是 `0.1` 默认推荐栈。

含义：

- 可以继续保留在仓库和文档里
- 可以有专项脚本、演练或烟测
- 行为、兼容性和运维语义仍允许在 `0.1.x` 到 `0.2` 之间继续收口
- 不作为首页默认生产部署建议

### Experimental

仅表示“代码路径存在”，不表示正式发布承诺。

含义：

- 不承诺兼容性、稳定性或恢复语义
- 不要求覆盖完整的发布测试矩阵
- 文档中必须显式标注，不应伪装成正式能力

## 0.1 GA 范围

`0.1` 当前只定义一条默认推荐栈：

- 数据库：PostgreSQL
- 缓存：Dragonfly
- 存储：本地目录
- 部署模型：基于仓库根目录 `compose.yml` 的单栈部署

这条 GA 主链路至少应覆盖：

- 数据库引导与安装向导
- 管理员创建、登录、刷新、登出、改密
- 图片上传、列表、过期时间管理、永久删除
- 结构化设置页与运行时设置持久化
- 审计日志、健康检查、系统统计
- PostgreSQL 物理备份、恢复与 PITR 运维主路径

## 0.1 Beta 范围

以下能力保留在 `0.1` 文档和仓库中，但按 Beta 处理：

- PostgreSQL + Dragonfly + S3 / MinIO
- MySQL 8.4 部署、烟测、运维脚本与演练
- MariaDB 12 部署、烟测、运维脚本与演练

Beta 的含义不是“不能用”，而是：

- 不作为 `0.1` 首页默认推荐栈
- 发布验收优先级低于 PostgreSQL GA 主链路
- 允许继续在后续阶段收紧行为边界

## 0.1 Experimental 范围

以下能力保留，但按 Experimental 处理：

- 无外部缓存模式
- 子路径部署
- “所有数据库后端都具备同等级恢复体验”的假设

这些能力在 `0.1` 周期内不应被描述为正式承诺，也不应成为发布阻塞项，除非后续阶段主动提升支持级别。

## 明确非目标

以下内容不属于 `0.1` 发布承诺：

- 对所有 `数据库 x 缓存 x 存储` 组合提供 GA 级保证
- MySQL / MariaDB / PostgreSQL 的页面内恢复
- 多节点、高可用、跨实例协调
- 零停机恢复、零停机迁移
- 对对象存储内容提供完整中间时间点回放
- 任意反向代理子路径下的完整前后端兼容承诺

## 阶段 0 退出标准

阶段 0 完成时，应满足：

- README 已链接并摘要这份支持范围
- PostgreSQL / MySQL 文档都引用这份支持级别定义
- 文档里不再出现互相冲突的“正式支持”表述
- 后续阶段的测试、CI 和发布门槛都以这份范围为依据

## 阶段 1 退出标准

阶段 1 完成时，应满足：

- 默认 PostgreSQL Compose smoke 已作为 `GA` 主链路门禁稳定运行
- PostgreSQL 物理备份 / 恢复演练与 PITR 演练都有仓库内统一入口
- 仓库已提供本地与 GitHub Actions 同构的 `release-ga-gate`
- 正式版发布阻塞项已经收口到 [`release-0.1-ga-checklist.md`](release-0.1-ga-checklist.md)

## 阶段 2 退出标准

阶段 2 完成时，应满足：

- 工作区版本、changelog 和运行时版本展示已经冻结到同一个候选版
- Docker 镜像已经带上可追溯的 version / revision / created 元数据
- 仓库已提供本地与 GitHub Actions 同构的 `release-rc-preflight`
- 候选版预检已经收口到 [`release-0.1-rc-runbook.md`](release-0.1-rc-runbook.md)

## 阶段 3 退出标准

阶段 3 完成时，应满足：

- 工作区版本和 changelog 已从候选版提升到稳定正式版号
- 仓库已提供本地与 GitHub Actions 同构的 `release-ga-ship`
- 发布说明、镜像元数据、bundle 和 `SHA256SUMS` 已能通过统一入口生成
- 正式版 cutover 已收口到 [`release-0.1-ga-runbook.md`](release-0.1-ga-runbook.md)
